\begin{abstract}
ByteVault is a distributed core banking application designed to demonstrate the feasibility of geographically distributed ledger systems without sacrificing strict ACID compliance. The project integrates a React-based frontend (Next.js) with a high-performance backend (Bun + Express) that orchestrates financial transactions across multiple independent PostgreSQL databases representing discrete bank branches. By leveraging PostgreSQL Foreign Data Wrappers (FDW) for cross-branch read consistency and a Two-Phase Commit (2PC) protocol for atomic distributed transactions, ByteVault successfully models a double-entry accounting ledger that prevents double-spending, partial commits, and cross-node data anomalies. This report outlines the architectural rationale, database modeling, and fault-tolerance mechanisms implemented to meet enterprise financial standards.
\end{abstract}
