\section{Operational Notes}
\subsection{Applying migrations to existing volumes}
If DB volumes already exist, apply incremental scripts:
\begin{lstlisting}[style=bash]
docker compose exec -T branch_a_db psql -U admin_a -d branch_a_db < docker/apply-security.sql
docker compose exec -T branch_b_db psql -U admin_b -d branch_b_db < docker/apply-security.sql

docker compose exec -T branch_a_db psql -U admin_a -d branch_a_db < docker/apply-ledger.sql
docker compose exec -T branch_b_db psql -U admin_b -d branch_b_db < docker/apply-ledger.sql

docker compose exec -T branch_a_db psql -U admin_a -d branch_a_db < docker/apply-holds.sql
docker compose exec -T branch_b_db psql -U admin_b -d branch_b_db < docker/apply-holds.sql

cat docker/apply-transfer-holds.sql | docker compose exec -T branch_a_db psql -U admin_a -d branch_a_db
cat docker/apply-transfer-holds.sql | docker compose exec -T branch_b_db psql -U admin_b -d branch_b_db

cat docker/apply-reversals.sql | docker compose exec -T branch_a_db psql -U admin_a -d branch_a_db
cat docker/apply-reversals.sql | docker compose exec -T branch_b_db psql -U admin_b -d branch_b_db
\end{lstlisting}

\subsection{Compliance (KYC) Dashboard}
The system tracks the Know Your Customer (KYC) status for all users. Statuses include:
\begin{itemize}
    \item \textbf{PENDING}: Initial state for new users.
    \item \textbf{VERIFIED}: User is cleared for full banking operations.
    \item \textbf{REJECTED}: User is blocked from opening new accounts or transacting.
\end{itemize}
These statuses are displayed as visual badges in the \texttt{AccountsTable} on the employee dashboard.

\subsection{Operational Audit Trail}
Every sensitive action in ByteVault is captured in the \texttt{audit\_logs} table. This includes:
\begin{itemize}
    \item Account creations and status changes.
    \item Transfer requests and approvals.
    \item Manual ledger adjustments.
\end{itemize}
Admin users can view the full history at \texttt{/admin/audit}, which displays the actor UUID, action type, entity involved, and associated metadata.
