\section{How to Run (Development)}
\subsection{Databases}
From repo root:
\begin{lstlisting}[style=bash]
docker compose up -d
docker compose ps
\end{lstlisting}

\subsection{Backend}
\begin{lstlisting}[style=bash]
cd backend
bun install
cp .env.example .env
bun run dev
\end{lstlisting}

\subsection{Client}
\begin{lstlisting}[style=bash]
cd client
bun install
cp .env.example .env.local
bun run dev
\end{lstlisting}
\subsection{Onboarding and Seeding}
Once the databases and backend are running, you must populate the system with initial records.

\subsubsection{Onboard an Employee}
To access the employee dashboard, create an admin or maker account:
\begin{lstlisting}[style=bash]
cd backend
bun run onboard --email admin@bytevault.com --password securepass --role ADMIN --branch MAIN --name "Admin User"
\end{lstlisting}

\subsubsection{Seed Demo Data}
To quickly populate the system with a sample user, bank account, and audit logs for testing:
\begin{lstlisting}[style=bash]
cd backend
bun run seed-demo
\end{lstlisting}
This command creates a verified user (\texttt{john.doe@example.com}) with an active account and an initial balance of ₹50,000.00.
